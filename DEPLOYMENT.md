# DNS Monitoring - Deployment Guide

A comprehensive guide for deploying DNS Monitoring anywhere: Unraid, VPS, Cloud providers, home servers, etc.

## Quick Start (Recommended)

```bash
# Clone repository
git clone <your-repo>
cd dns-monitoring

# Make scripts executable
chmod +x deploy.sh backup.sh restore.sh ssl-setup.sh

# Run interactive deployment
./deploy.sh

# Answer the prompts and you're done!
```

## What You Get

After running `./deploy.sh`, you'll have:

✅ DNS Monitoring running in Docker
✅ Automatic data persistence
✅ Health checks enabled
✅ Reverse proxy with Nginx configured
✅ Environment variables properly set
✅ `.env.prod` file for your configuration
✅ Ready to access via your domain/IP

## Deployment Scenarios

### Scenario 1: Unraid Server

**Prerequisites:**
- Unraid with Docker installed
- SSH or terminal access

**Steps:**

```bash
# SSH into Unraid
ssh root@unraid-ip

# Clone repository
cd /mnt/user/appdata
git clone <your-repo>
cd dns-monitoring

# Make scripts executable
chmod +x deploy.sh

# Run deployment
./deploy.sh

# Choose:
# 1) IP address
# Enter: your-unraid-ip (e.g., 192.168.1.50)
# Enter: http://192.168.1.50/api
```

**Access:** `http://192.168.1.50` from any computer on your network

---

### Scenario 2: DigitalOcean / Linode / Hetzner VPS

**Prerequisites:**
- Ubuntu 20.04+ or Debian 10+ server
- Docker installed: `curl -fsSL https://get.docker.com | sh`
- Domain name (with DNS pointing to VPS IP)

**Steps:**

```bash
# SSH into VPS
ssh root@your-vps-ip

# Install Docker Compose (if not included)
sudo apt update && sudo apt install -y docker-compose

# Clone repository
git clone <your-repo>
cd dns-monitoring

# Make scripts executable
chmod +x deploy.sh ssl-setup.sh

# Run deployment
./deploy.sh

# Choose:
# 2) Domain name
# Enter: yourdomain.com
# Use HTTPS? y
```

**Setup HTTPS (after deployment):**

```bash
# Generate SSL certificate
./ssl-setup.sh yourdomain.com

# Restart with HTTPS
docker-compose -f docker-compose.prod.yml restart nginx
```

**Access:** `https://yourdomain.com`

---

### Scenario 3: Home Server / NAS

**Prerequisites:**
- Docker available
- Network access to your home

**Steps:**

```bash
# SSH into your server
ssh user@192.168.1.100

# Clone repository
git clone <your-repo>
cd dns-monitoring

# Make scripts executable
chmod +x deploy.sh

# Run deployment
./deploy.sh

# Choose:
# 1) IP address
# Enter: 192.168.1.100
# Enter: http://192.168.1.100/api
```

**External Access (Optional):**

Use a reverse proxy:
- **Cloudflare Tunnel** (easiest, free)
- **Tailscale** (private network)
- **ngrok** (temporary, development)
- **Port forwarding** (risky, not recommended)

**Access:** `http://192.168.1.100` or via tunnel

---

### Scenario 4: Raspberry Pi

**Prerequisites:**
- Raspberry Pi 3B+ or better
- Raspberry Pi OS installed
- Docker: `curl -fsSL https://get.docker.com | sh`

**Steps:**

```bash
# SSH into Pi
ssh pi@raspberrypi.local

# Add user to docker group
sudo usermod -aG docker $USER
newgrp docker

# Clone repository
git clone <your-repo>
cd dns-monitoring

# Make scripts executable
chmod +x deploy.sh

# Run deployment (may be slow on Pi)
./deploy.sh

# Answer prompts and wait (be patient!)
```

**Note:** RPi 2 may not work. Use RPi 3B+ or better.

## Configuration Files

### `.env.prod` (Auto-generated by deploy.sh)

Contains your deployment-specific settings:

```bash
# After running ./deploy.sh, edit .env.prod to customize:
VITE_API_URL=http://192.168.1.100/api      # Frontend API URL
CORS_ORIGIN=http://192.168.1.100           # Backend CORS origin
DEFAULT_DNS_SERVER=1.1.1.1                 # Default DNS server
TZ=Asia/Singapore                          # Timezone
QUERY_RETENTION_DAYS=30                    # Data retention
```

### `docker-compose.prod.yml`

Main deployment configuration. Usually no changes needed—all customization via `.env.prod`.

### `nginx.conf`

Reverse proxy settings. Edit only if you need custom routing or SSL.

## Common Operations

### View Logs

```bash
# All services
docker-compose -f docker-compose.prod.yml logs -f

# Backend only
docker-compose -f docker-compose.prod.yml logs -f backend

# Nginx only
docker-compose -f docker-compose.prod.yml logs -f nginx

# Last 100 lines
docker-compose -f docker-compose.prod.yml logs --tail=100
```

### Stop / Start Services

```bash
# Stop all services
docker-compose -f docker-compose.prod.yml down

# Start services
docker-compose -f docker-compose.prod.yml up -d

# Restart services
docker-compose -f docker-compose.prod.yml restart

# Restart backend only
docker-compose -f docker-compose.prod.yml restart backend
```

### Update to Latest Version

```bash
# Pull latest image
docker pull ghcr.io/freemen810/dns-monitoring:latest

# Restart with new image
docker-compose -f docker-compose.prod.yml up -d

# Verify
curl http://your-domain/api/health
```

### Check Health

```bash
# Full status
docker-compose -f docker-compose.prod.yml ps

# Health check only
curl http://your-domain/api/health
# Response: {"status":"ok"}
```

### View Database

```bash
# Access SQLite directly
docker-compose -f docker-compose.prod.yml exec backend sqlite3 /app/data/dns_monitor.db

# List monitors
sqlite> SELECT id, name, domain FROM monitors;

# Exit
sqlite> .exit
```

## Backup & Restore

### Create Backup

```bash
# Automatic backup (with cleanup)
./backup.sh

# Backup to custom location
./backup.sh /mnt/backup
```

**Backups stored:** `backups/dns-monitor-backup-YYYYMMDD-HHMMSS/`

### Restore Backup

```bash
# Interactive restore
./restore.sh backups/dns-monitor-backup-20260321-120000

# Confirm when prompted
# Automatic safety backup created before restore
```

### Automated Backups

```bash
# Add to crontab (runs daily at 2 AM)
crontab -e

# Add this line:
0 2 * * * cd /path/to/dns-monitoring && ./backup.sh >> backups/backup.log 2>&1
```

## SSL/HTTPS Setup

### Automatic Setup (Recommended)

```bash
# Interactive SSL setup
./ssl-setup.sh

# Or non-interactive
./ssl-setup.sh yourdomain.com your-email@example.com

# Updates:
# 1. Generates free Let's Encrypt certificate
# 2. Copies to ssl/ directory
# 3. Sets up auto-renewal
```

### Manual Setup

```bash
# Create ssl/ directory
mkdir -p ssl

# Copy your own certificates
cp /path/to/cert.pem ssl/
cp /path/to/key.pem ssl/

# Set permissions
chmod 644 ssl/*.pem

# Uncomment SSL lines in docker-compose.prod.yml
# Restart nginx
docker-compose -f docker-compose.prod.yml restart nginx
```

## Troubleshooting

### Containers Won't Start

```bash
# Check logs
docker-compose -f docker-compose.prod.yml logs

# Check Docker daemon
docker info

# Restart Docker
sudo systemctl restart docker

# Full rebuild
docker-compose -f docker-compose.prod.yml down
docker pull ghcr.io/freemen810/dns-monitoring:latest
docker-compose -f docker-compose.prod.yml up -d
```

### High Memory Usage

```bash
# Check container stats
docker stats

# Limit memory in docker-compose.prod.yml:
deploy:
  resources:
    limits:
      memory: 512M
```

### Database Locked

```bash
# SQLite lock (rare)
docker-compose -f docker-compose.prod.yml exec backend rm -f /app/data/*.db-wal

# Restart
docker-compose -f docker-compose.prod.yml restart backend
```

### API Returns 503 / 502

**Backend crashed:**
```bash
docker-compose -f docker-compose.prod.yml logs backend
docker-compose -f docker-compose.prod.yml restart backend
```

**Nginx issue:**
```bash
docker-compose -f docker-compose.prod.yml logs nginx
docker-compose -f docker-compose.prod.yml exec nginx nginx -t
```

### Slow Performance

```bash
# Check system resources
docker stats

# Monitor database size
du -sh dns-data/dns_monitor.db

# Check logs for errors
docker-compose -f docker-compose.prod.yml logs backend | tail -20
```

## Advanced Configuration

### Custom Port

```bash
# Edit .env.prod
EXTERNAL_PORT=8080

# Restart
docker-compose -f docker-compose.prod.yml up -d
```

### Custom Timezone

```bash
# Edit .env.prod
TZ=America/New_York

# Restart
docker-compose -f docker-compose.prod.yml restart
```

### Behind Reverse Proxy (Apache/Nginx)

If you're running your own reverse proxy, configure it to forward to localhost:

**Nginx:**
```nginx
server {
    listen 443 ssl http2;
    server_name yourdomain.com;

    location / {
        proxy_pass http://localhost:80;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

**Apache:**
```apache
<VirtualHost *:443>
    ServerName yourdomain.com
    ProxyPreserveHost On
    ProxyPass / http://localhost:80/
    ProxyPassReverse / http://localhost:80/
</VirtualHost>
```

### Multiple Instances

```bash
# Deploy another instance with different port
CONTAINER_PREFIX=dns-monitor-2 EXTERNAL_PORT=8080 ./deploy.sh
```

## Security Best Practices

### 1. Use HTTPS

```bash
./ssl-setup.sh yourdomain.com
```

### 2. Restrict CORS

Edit `.env.prod`:
```bash
# Instead of:
CORS_ORIGIN=*

# Use:
CORS_ORIGIN=https://yourdomain.com
```

### 3. Firewall Rules

```bash
# Allow only needed ports
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 22/tcp    # SSH
sudo ufw deny 3000/tcp   # Backend not exposed
```

### 4. Regular Backups

```bash
# Automated daily backups
crontab -e
# Add: 0 2 * * * cd /path/to/dns-monitoring && ./backup.sh
```

### 5. Keep Updated

```bash
# Check for updates weekly
docker pull ghcr.io/freemen810/dns-monitoring:latest

# Apply updates
docker-compose -f docker-compose.prod.yml up -d
```
